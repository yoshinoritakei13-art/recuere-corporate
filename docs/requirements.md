# recuere コーポレートサイト - 要件定義書

## 要件定義の作成原則
- **「あったらいいな」は絶対に作らない**
- **拡張可能性のための余分な要素は一切追加しない**
- **将来の「もしかして」のための準備は禁止**
- **今、ここで必要な最小限の要素のみ**

---

## 1. プロジェクト概要

### 1.1 ブランドコンセプト

**ブランド名**: recuere（リキュウレ）

**商標表記ルール**:
- 「recuere」「リキュウレ」は両方とも商標登録済み（®使用可能）
- **ロゴ・会社名表記**: `recuere`（®不要）
- **コピーライト**: `© 2025 recuere Inc.`（®不要）
- **文章中の初出**: `recuere®` / `リキュウレ®`（各ページ最初のみ）
- **2回目以降**: `recuere` / `リキュウレ`（®不要）
- **全て小文字で統一**（Recuere、RECUERE は使用しない）

**ブランドキーワード**:
- 気づき
- 豊かさ
- 本来の自分に戻る
- 静けさ
- 透明感
- やわらかな未来感
- 内側の変化が外側へ波紋のように広がる

**ビジュアル方向性**:
- 白を基調とした透明空間
- 水色、薄紫、朝焼けオレンジを1滴
- 過度に華やかにせず、静かで上質
- 「スピリチュアル」ではなく「ミニマル × 心理 × 組織」
- 揺らぎは優しく、呼吸のように

**禁忌表現**:
- 派手な虹色
- 強いグラデ
- 濃い線
- 占い・幻想系に見える表現
- 暗い背景
- 幾何学模様の機械感

### 1.2 サイトの目的

**対象**:
- メイン: 企業経営者・医療法人／歯科医院の経営層・人事／組織開発担当
- サブ: 個人のクライアント（自己実現、ヒプノ、ブランディングなど）

**サイトのゴール**:
1. recuereの世界観（「気づき」「豊かさ」「本来の自分に戻る」）を伝えるブランディング
2. 企業向け・個人向けともに「お問い合わせ／相談」の獲得
3. 特に企業向けコンサル・研修の信頼感を高めることを優先

### 1.3 提供サービス

**企業向け**:
- 経営コンサルティング
- 医療法人・歯科医院向けコンサルティング
- プロジェクト遂行支援
- 人材育成研修（営業・マネジメント・育成系）

**個人向け**:
- 自己実現セッション（カウンセリング／コーチング／潜在意識リーディング）
- ヒプノセラピー
- パーソナルコーチング（継続）
- ブランディング構築セッション

### 1.4 成功指標

#### 定量的指標
- ページ読み込み速度: LCP 2.5秒以内
- モバイル対応: Lighthouse モバイルスコア90点以上
- ページ数: 6ページ + 補助1ページ
- 流動リボンアニメーション: パフォーマンス阻害なし
  - スマホで滑らかに動くこと最優先
  - long task（50ms以上）を発生させない
  - prefers-reduced-motion対応

#### 定性的指標
- fivot.co.jp風の「透明感」「余白」「流線の美しさ」を表現
- 流動リボンは「呼吸するようなゆっくりした揺らぎ」
- 医療法人・企業担当者が「信頼できる」「任せられそう」と感じるトーン
- 「ミニマル × 心理 × 組織」の世界観（スピリチュアル寄りにしない）

### 3.6 P-006: Contact（/contact）

#### 目的
企業用/個人用フォームで問い合わせを獲得。コンバージョンポイント。

#### フォーム構成

```yaml
共通項目:
  - お問い合わせ種別（企業向け / 個人向け）: プルダウン選択

企業向け選択時:
  - 会社名（必須）
  - 担当者名（必須）
  - メールアドレス（必須）
  - 電話番号（任意）
  - お問い合わせ内容（必須）

個人向け選択時:
  - お名前（必須）
  - メールアドレス（必須）
  - お問い合わせ内容（必須）
```

#### 処理フロー
1. ユーザーが問い合わせ種別を選択
2. 種別に応じたフォーム項目を表示
3. フォームに入力
4. reCAPTCHA v3でスコア取得
5. スコア閾値を超えた場合のみ送信
6. Resendでメール送信（送信先は種別により振り分け）
7. 成功時: /contact/thanks へリダイレクト
8. 失敗時: エラーメッセージ表示

## 4. デザイン仕様

### 4.1 カラーパレット

```yaml
背景:
  - 白に近い淡青ホワイト
  - ごく弱い縦方向グラデ（上：少し暖かい／下：少し冷たい）

リボンカラー:
  - 水色（メイン 60%）
  - 青〜ラベンダーの薄い紫（サブ 30%）
  - 朝焼けオレンジの光（差し色 10%）
  - 彩度は低め、透明重視

テキスト:
  - ダークグレー（#333 程度）
  - 控えめで読みやすい
```

### 4.2 流動リボンアニメーション仕様

```yaml
役割: トップページHERO背景の主役演出

構図:
  - 画面左下から右上へ向かって大きく緩やかなS字カーブ
  - 複数本（2〜3）の帯が重なり、奥行きがある
  - fivot系だが完全一致は避ける（カーブ位置は変更）

質感:
  - 極めて薄く透明度の高い光の帯
  - 内部に多数の極細ライン（1.5〜2px）を含む
  - エッジはシャープにしすぎず、soft blurで発光
  - 重なった部分は光がわずかに強まる（影は不要）

雰囲気:
  - 透明、静か、未来的
  - 空気の中に光が漂う印象
  - 「気づき→広がる流れ」というテーマが伝わる柔らかさ

実装方式:
  - 軽量なCSS/SVG＋簡易JSアニメーション または Lottie
  - WebGLは不要（重い実装は避ける）
  - スマホで滑らかに動くこと最優先

技術要件:
  - dynamic import（ssr: false）
  - IntersectionObserverでの遅延描画
  - prefers-reduced-motion で停止 or 静止
  - pointer-events: none（クリックを邪魔しない）

ループ: ON（15〜20秒/ループ程度、ゆっくり）
透明度: 0.2〜0.3（控えめ）
```

### 4.3 タイポグラフィ

```yaml
見出し:
  - 大きく、余白をたっぷり
  - 日本語: システムフォント or Noto Sans JP
  - 英語: Inter or システムフォント

本文:
  - 読みやすいサイズ（16px以上）
  - 行間ゆったり（1.8〜2.0）

全体:
  - 控えめ、静か
  - 文字が主役、装飾は最小限
```

### 4.4 参考サイト

```yaml
デザイン・世界観のメイン参考:
  - fivot（https://fivot.co.jp/）
    - 白背景＋流線グラデーション
    - トップの光のリボンの雰囲気
    - 余白の取り方・フォント感

動きや質感の参考:
  - tokyotechnology.jp
    - 有機的で滑らかな揺らぎ方

オリジナル要素:
  - 朝焼けのオレンジを少し混ぜる
  - 「ミニマル × 心理 × 組織」の世界観に寄せる
  - 形やカーブはオリジナル
```
